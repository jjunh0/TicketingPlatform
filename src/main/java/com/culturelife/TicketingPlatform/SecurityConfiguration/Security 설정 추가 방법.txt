
1. SecurityConfiguration의 중간 부분에 추가해주셔야 하는 부분이 있습니다.
    해당 클래스에 주석을 작성해 놓았으니 해당 설명을 따라 진행해주시면 됩니다.
    위치는 SecurityConfiguration 패키지의 SecurityConfig 클래스입니다.

2. 프로그램 실행 시에, h2 db에 초기값이 생성되도록 설정하였습니다.

    아이디 admin, 비밀번호 123 - ADMIN 권한 유저 (관리자에 해당)
    아이디  user, 비밀번호 123 -  USER 권한 유저 (일반 유저에 해당)

    권한에 따른 테스트가 필요하신 경우, 해당 아이디로 로그인하신 후 진행하시면 됩니다.

3. 테스트를 위한 /test1, /test2 페이지가 있습니다.

    /test1은 ADMIN 권한 사용자만 접근 가능하며,
    /test2는 ADMIN, USER 권한 사용자 모두 접근 가능합니다.

    해당 페이지로 접근하면 현재 로그인된 사용자의 id(고유값), 아이디, 이메일 등 패스워드를 제외한 정보를 확인할 수 있습니다.

4. 그래서 지금 로그인되어 있는 유저에 대한 정보가 필요한데, 어떻게 받아오나요?

    Service 패키지에 있는 AuthenticatedUserService 클래스에서,
    현재 로그인된 유저의 id, memberId, roles(권한)을 가져올 수 있는 메서드를 만들어 놓았습니다.
    메서드의 리턴값은 코드를 보셔도 되고, /test1이나 /test2에 접속하셔서 값을 확인해도 됩니다.
    확인하신 걸 바탕으로 본인 파트 작업을 진행해주시면 됩니다.
